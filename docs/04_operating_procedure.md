# Operating Procedure — Magnetic Field Nulling

**Status**: Draft framework. Steps 3 onwards require hardware installation and characterisation to be complete.

This procedure allows an operator to null the residual DC magnetic field and bring both QuSpin sensors into their linear operating range. It must be executable by someone other than the original engineer.

---

## Prerequisites

Before starting, confirm:

- [ ] Both QuSpin sensors powered and warmed up (**[TBD — warm-up time from datasheet]**)

- [ ] All three current setpoints at zero (or known safe starting values)
- [ ] `coil_calibration.json` present in `data/processed/` (generated by `scripts/coil_cal/calibrate.py`)

## Nulling procedure

### Step 1: Confirm saturation baseline

Observe the sensor output. Both sensors should show flat lines (saturated). If either sensor is already dynamic, it is not saturated — record this and skip to Step 4.

### Step 3: Axis-by-axis trim

Work one axis at a time. The axis with the largest estimated residual field should be trimmed first.

**For each axis** (repeat for X, Y, Z in order of decreasing estimated residual):

1. Slowly increase (or decrease) the current on the selected axis.
2. Observe the sensor output — watch for the flat line to break and the signal to become dynamic.
3. Once one or both sensors show dynamic output, fine-trim the current until the output is centred near zero field (minimum signal amplitude for a DC field).
4. Record the setpoint in mA.
5. Move to the next axis.

**Notes**:
- Adjusting one axis may shift the apparent optimum on another axis due to coil cross-coupling. One iteration over all three axes is usually required; repeat if needed.
- Cross-coupling correction matrix: **[TBD — from coil calibration, if significant]**
- If sensors saturate again after adjusting a second axis, return to the first axis and re-trim.

---

### Step 4: Confirm null

1. Record 60 s of data with all setpoints fixed.
2. Confirm both sensors show dynamic (non-railed) output.
3. Run `scripts/nulling/check_null.py` — reports estimated residual field per sensor and confirms pass/fail against REQ-SYS-001.

---

### Step 5: Record and save

Record the following in the lab notebook and in `data/raw/YYYY-MM-DD_null_setpoints.json`:

```json
  "residual_field_nT": {"sensor1": 0.0, "sensor2": 0.0},
  "notes": ""
}
```

## Troubleshooting

| Symptom | Likely cause | Action |
|---------|-------------|--------|
| Sensors do not exit saturation at any coil current | Coil authority insufficient (REQ-COIL-001 violated), or wrong axis | Check coil connections, verify B/I calibration, try reversing current direction |
| One sensor exits saturation but not the other | Field gradient exceeds REQ-COIL-002 | See Risk C-01 in `02_design/coil_system.md` |
| Sensors exit saturation but immediately re-saturate | Cross-axis coupling | Complete all three axes before assessing; iterate |
| Setpoints drift over time — sensors re-saturate after 1-2 h | Current source drift (REQ-SYS-003 violated) | See `02_design/drive_electronics.md` drift budget |
| Spectral peaks appear after connecting current source | Switching noise injection | See Risk D-01 and VT-009 |

## Re-nulling

If the system needs to be re-nulled (e.g. after power cycle, equipment move, or thermal excursion):

1. Load previous setpoints from `data/raw/YYYY-MM-DD_null_setpoints.json` as a starting point.
2. Apply those currents.
3. Check sensor output — if sensors are already in linear range, no further action needed.
4. If not, fine-trim per Step 3 above.
5. Save new setpoints.

Previous setpoints are a good starting point but are not guaranteed to transfer between sessions if the field environment has changed (e.g. equipment rearranged in the lab, temperature shift).
